# 测试策略（首版）

## 测试目标

- 核心功能可用：登录认证、权限控制、审核流、搜索。
- 关键接口稳定：输入校验、错误码一致、鉴权正确。
- 发布前可快速回归：自动化测试 + 冒烟测试。

## 测试分层

1. 单元测试（后端）

- 使用 Pytest。
- 覆盖服务层与关键工具函数。

2. 接口测试（后端）

- 覆盖登录、资料编辑、审核、搜索接口。
- 验证正常流、权限拒绝流、异常流。

3. 前端验证

- 基础页面交互与表单校验。
- 联调时重点验证状态切换与错误提示。

## 最低覆盖建议

- 后端核心模块覆盖率建议 >= 80%。
- 新增业务接口必须至少包含 1 个成功用例与 1 个失败用例。

## 发布前检查清单

1. 前端 typecheck 通过。
2. 前端 build 通过。
3. 后端测试通过。
4. 至少完成一轮关键路径冒烟：注册 -> 登录 -> 编辑 -> 提审 -> 审核 -> 搜索。
