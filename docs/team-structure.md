# 🧑‍🤝‍🧑 名人堂百科项目分工说明（最终版）

## 一、总体说明

本项目采用前后端分离架构（React + FastAPI + SQLite + Docker），团队共 10 人。

本次为最终分工调整：

- ❌ 移除：文档专职 / 测试专职 / 全栈辅助 / 综合支持
- ✅ 前端扩展为 **4 人**
- ✅ 所有成员均参与核心开发

> ⚠️ 原则：**减少非核心角色，强化开发主链路，提高推进效率**

---

## 二、角色划分与职责（最终结构）

### 👑 1. 总负责人（1人）

**职责：**

- 技术架构设计与开发规范制定
- 主导 API 设计与接口统一
- 拆分任务与控制整体进度
- 前后端联调与问题排查
- 代码 Review（关键模块必须参与）
- 负责部署（Docker + Nginx）与测试策略

---

### 🧑‍💼 2. 项目经理 / 技术顾问（1人）

**职责：**

- 跟踪项目进度与任务执行情况
- 组织例会（每周至少 2 次）
- 管理需求优先级
- 协助测试与验收流程（无专职测试）

---

### 🗄️ 3. 数据库工程师（1人）

**职责：**

- 设计数据库 ER 图（必须在第一阶段完成并定稿）
- 核心数据表设计：
  - User（用户）
  - Profile（履历）
  - Review（审核）
  - Draft（草稿）

- ORM 建模（SQLAlchemy / SQLModel）
- 协助后端优化数据结构与查询性能

---

### ⚙️ 4-5. 后端工程师（2人）

#### 后端 A（核心系统）

**职责：**

- 用户系统（注册 / 登录 / JWT）
- 权限控制（管理员 / 普通用户）
- 审核流程（状态机，核心难点）

---

#### 后端 B（功能模块）

**职责：**

- 搜索接口（多条件查询）
- 文件上传（图片 / PDF）
- 草稿箱逻辑
- 日志系统与异常处理

---

👉 要求：

- 接口风格统一（由总负责人把控）
- 必须编写接口文档（Swagger / 注释）

---

### 💻 6-9. 前端工程师（4人）

#### 前端 A（基础框架 / UI）

- 首页 UI
- 全局布局（导航栏、主题切换）
- TailwindCSS 规范

---

#### 前端 B（用户系统）

- 登录 / 注册页面
- 用户信息展示
- 表单交互逻辑

---

#### 前端 C（核心功能）

- 搜索页面
- 搜索结果展示
- 多条件筛选 UI

---

#### 前端 D（内容与管理）

- 个人资料页
- 草稿箱 / 编辑页
- 管理员审核页面

---

👉 要求：

- 统一组件规范
- 前端运行时在 Bun 或 Deno 中二选一（默认优先 Bun），不使用传统 Node.js 作为开发运行时
- 使用 TypeScript 类型约束，不新增 JavaScript 业务文件
- 每次提交前通过 typechecker，发布前通过 build
- 避免重复组件开发

---

### 🚀 10. DevOps / 部署工程师（1人）

**职责：**

- 编写 Dockerfile
- 配置 docker-compose
- 配置 Nginx 反向代理
- 协助项目上线部署（在总负责人指导下）

---

## 三、阶段性任务分配

### 第一阶段（第1-2周）：基础搭建（最关键）

- 数据库工程师：完成 ER 图 + 表结构（必须一次性确定）
- 总负责人 + 后端：制定 API 文档
- 前端（4人）：完成页面原型设计（首页 / 搜索 / 个人页）
- DevOps：准备基础部署环境（Docker 初版）

---

### 第二阶段（第3-4周）：基础功能开发

- 后端 A：登录注册 + JWT
- 后端 B：基础 CRUD + 草稿箱
- 前端：
  - A：页面框架
  - B：登录系统
  - C：搜索基础 UI
  - D：个人页面基础结构

---

### 第三阶段（第5-6周）：核心功能（项目核心）

- 后端 A：审核流程（核心）
- 后端 B：搜索接口
- 前端：
  - C：搜索功能完善
  - D：管理员审核页面

- 全员：开始联调

---

### 第四阶段（第7-8周）：优化与扩展

- 后端：性能优化 / Redis（可选）
- 前端：UI优化与交互优化
- 可选功能开发（AI搜索 / 推荐）

---

### 第五阶段（第9-10周）：测试与部署

- 总负责人：整体联调 + 最终验收
- DevOps：部署上线
- 全员：Bug修复与自测

---

## 四、协作规范（必须遵守）

### 1. Git 规范

- ❌ 禁止直接 push main
- ✅ 必须通过 PR 合并

---

### 2. 开发原则

- 接口先行（先定义 API 再开发）
- 小步提交（避免大规模修改）
- 每周至少进行一次联调
- 前端 PR 合并前必须通过 TypeScript 类型检查（`bun run typecheck` 或 `deno task typecheck`）
- 前端上线前必须完成构建（`bun run build` 或 `deno task build`）

---

### 3. 测试原则（重要）

- 不设专职测试人员
- 👉 每个开发者必须为自己模块负责测试
- 👉 核心功能必须重点验证：
  - 登录认证
  - 权限控制
  - 审核流程

---

本项目最终组织结构遵循：

> **以“前后端开发”为绝对核心，压缩辅助角色，让所有成员直接参与核心产出。**

该结构适用于：

- 小团队
- 新手团队
- 快速推进型项目
