# 系统架构说明

本项目采用前后端分离架构：React + FastAPI + SQLite（可选 Redis）+ Nginx + Docker。

## 架构分层

1. 展示层（Frontend）

- React + TypeScript 页面与组件
- 通过 REST API 与后端通信

2. 应用层（Backend API）

- FastAPI 路由、鉴权、参数校验
- 业务逻辑：用户、百科、审核、搜索、草稿

3. 数据层（Database）

- SQLite 持久化核心业务数据
- ORM 管理模型与查询
- Redis 仅在并发压力出现后按需接入

4. 交付层（Ops）

- Nginx 做反向代理和静态资源分发
- Docker / Compose 管理部署

## 关键业务流

1. 注册登录：用户注册 -> 登录 -> 获取 JWT -> 持 token 请求受保护接口。
2. 百科编辑：用户编辑资料 -> 保存草稿 -> 提交审核 -> 管理员审批。
3. 搜索检索：前端输入关键词 -> 后端多字段查询 -> 返回分页结果。

## 非功能要求

- 安全：权限校验、输入校验、上传限制、敏感信息脱敏。
- 性能：分页查询、索引优化、必要时引入缓存。
- 可维护：接口先行、文档先行、迁移可回滚。
