# 名人堂百科项目分工说明（迁移版）

> 本文由 `docs/team-structure.md` 迁移而来，作为当前正式版本。

## 一、总体说明

本项目采用前后端分离架构（React + FastAPI + SQLite + Docker），团队共 10 人。

组织原则：减少非核心角色，强化开发主链路，提高推进效率。

## 二、角色划分与职责

### 1. 总负责人（1人）

- 技术架构设计与开发规范制定
- 主导 API 设计与接口统一
- 拆分任务与控制整体进度
- 前后端联调与问题排查
- 关键模块代码评审
- 负责部署策略（Docker + Nginx）与测试策略

### 2. 项目经理 / 技术顾问（1人）

- 跟踪进度与任务执行
- 组织例会（每周至少 2 次）
- 管理需求优先级
- 协助测试与验收流程

### 3. 数据库工程师（1人）

- 第一阶段完成 ER 图并定稿
- 设计 User、Profile、Review、Draft 等核心表
- 负责 ORM 建模（SQLAlchemy / SQLModel）
- 协助后端优化查询性能

### 4-5. 后端工程师（2人）

后端 A（核心系统）：

- 注册 / 登录 / JWT
- 权限控制（管理员 / 普通用户）
- 审核流程状态机

后端 B（功能模块）：

- 搜索接口（多条件查询）
- 文件上传（图片 / PDF）
- 草稿箱逻辑
- 日志系统与异常处理

统一要求：

- 接口风格统一（总负责人把控）
- 必须维护接口文档（OpenAPI / Swagger）

### 6-9. 前端工程师（4人）

前端 A（基础框架 / UI）：

- 首页 UI
- 全局布局（导航栏、主题切换）
- TailwindCSS 规范

前端 B（用户系统）：

- 登录 / 注册页
- 用户信息展示
- 表单交互逻辑

前端 C（核心功能）：

- 搜索页
- 搜索结果展示
- 多条件筛选 UI

前端 D（内容与管理）：

- 个人资料页
- 草稿箱 / 编辑页
- 管理员审核页

统一要求：

- 统一组件规范，避免重复开发
- 前端运行时在 Bun 或 Deno 二选一（默认 Bun）
- 必须使用 TypeScript，不新增 JavaScript 业务文件
- 提交前通过 typecheck，发布前通过 build

### 10. DevOps / 部署工程师（1人）

- 编写 Dockerfile
- 配置 docker-compose
- 配置 Nginx 反向代理
- 在总负责人指导下完成上线部署

## 三、阶段性任务分配

### 第一阶段（第1-2周）：基础搭建

- 数据库：ER 图 + 表结构定稿
- 总负责人 + 后端：API 文档定版
- 前端（4人）：首页 / 搜索 / 个人页原型
- DevOps：基础部署环境初版

### 第二阶段（第3-4周）：基础功能开发

- 后端 A：登录注册 + JWT
- 后端 B：基础 CRUD + 草稿箱
- 前端 A/B/C/D 分模块推进页面与交互

### 第三阶段（第5-6周）：核心功能

- 后端 A：审核流程
- 后端 B：搜索接口
- 前端 C/D：搜索完善 + 审核页
- 全员联调

### 第四阶段（第7-8周）：优化与扩展

- 后端：性能优化 / Redis（可选）
- 前端：UI 与交互优化
- 全员：按进度选择可选功能

### 第五阶段（第9-10周）：测试与部署

- 总负责人：联调与最终验收
- DevOps：部署上线
- 全员：缺陷修复与自测

## 四、协作规范

### 1. Git 规范

- 禁止直接 push main
- 必须通过 PR 合并

### 2. 开发原则

- 接口先行（先定义 API 再开发）
- 小步提交（避免大规模修改）
- 每周至少一次联调
- 前端 PR 前通过 typecheck
- 前端上线前完成 build

### 3. 测试原则

- 不设专职测试
- 每个开发者对自己模块测试负责
- 核心功能重点验证：登录认证、权限控制、审核流程
