# 数据库文档总览

本目录用于沉淀数据库设计与演进规范，确保表结构、关系和迁移策略有统一文档。

## 文件说明

- `schema.md`：核心表结构与字段约定
- `er-diagram.md`：实体关系与业务流说明
- `migration.md`：迁移、回滚与数据变更流程

## 设计原则

1. 优先保证数据可追溯：关键业务数据保留创建/更新时间与操作者信息。
2. 严格区分当前态与历史态：审核记录、草稿记录单独建表。
3. 上传文件只存元数据与路径，不将文件本体写入数据库。
4. 任何破坏性变更必须先在 `migration.md` 制定回滚方案。
