# 部署与运维指南（首版）

## 目标环境

- Linux 服务器（推荐）
- Docker / Docker Compose
- Nginx 反向代理

## 部署前检查

1. 环境变量准备：数据库路径、JWT 密钥、上传路径等。
2. 安全检查：确认 `.env` 未入库，仅保留 `.env.example`。
3. 构建检查：前端通过 typecheck 与 build，后端测试通过。

## 参考流程

1. 拉取代码并切换目标版本。
2. 生成镜像并启动容器。
3. 执行数据库迁移。
4. 执行健康检查和接口冒烟测试。
5. 切换流量并观察日志。

## 健康检查建议

- API 健康检查：`/api/v1/health`
- Nginx 状态页或访问日志监控
- 应用日志中错误率与响应时间观察

## 日志与备份

- 应用日志按级别输出（INFO/WARN/ERROR）
- 每日备份数据库并定期恢复演练
- 保留最近发布版本，支持快速回滚

## 故障处理

1. 先确认是应用问题还是基础设施问题。
2. 若为发布引入问题，优先回滚版本。
3. 保留现场日志并输出复盘结论。
