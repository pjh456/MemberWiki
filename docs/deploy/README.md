# 部署与运维指南

## 目标环境

- Linux 服务器（推荐）
- Docker / Docker Compose
- Nginx 反向代理

## 部署前检查

1. 准备环境变量：数据库路径、JWT 密钥、上传路径等。
2. 确认 `.env` 不入库，仅维护 `.env.example`。
3. 前端通过 typecheck 与 build，后端测试通过。

## 推荐发布流程

1. 拉取目标版本代码。
2. 构建镜像并启动服务。
3. 执行数据库迁移。
4. 执行健康检查和关键接口冒烟。
5. 观察日志后切换流量。

## 观测与日志

- 健康检查接口：`/api/v1/health`
- 应用日志按级别输出：INFO/WARN/ERROR
- 关注错误率、响应时间、鉴权失败次数

## 备份与回滚

- 每日全量备份数据库，保留至少 7 天。
- 发布前增加一次手动备份。
- 发布异常时优先回滚到上一个稳定版本并恢复备份。
